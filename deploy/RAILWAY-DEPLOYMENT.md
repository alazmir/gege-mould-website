# Option B: Split Deployment — Railway.app (Backend API)

> **Cost:** Free (Railway free tier)
> **Best for:** No upgrade needed — works with existing Hostinger Premium plan

---

## Overview

Since Hostinger Premium (shared hosting) cannot run Node.js, we split the stack:

| Component | Host | Cost |
|---|---|---|
| Frontend (HTML/CSS/JS) | Hostinger Premium | Already paid |
| Backend (Express API) | Railway.app | Free tier |

The `dist/` folder (generated by `node build.js`) contains pre-rendered frontend pages with all headers, footers, and scripts baked in — no server-side processing needed. The backend runs as a standalone API on Railway, handling form submissions and email notifications.

1. Go to [railway.app](https://railway.app/)

## Step 2: Prepare the Backend for Railway

The backend needs a few small adjustments for standalone API mode. Create a `railway.json` file in the project root:

```json
{
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "node backend/server.js",
    "healthcheckPath": "/api/health",
    "restartPolicyType": "ON_FAILURE"
  }
}
```

Railway auto-detects the Node.js project from `backend/package.json` and installs dependencies automatically.

## Step 3: Deploy to Railway

### Option A: Deploy via GitHub (Recommended)

1. Push this project to a **private** GitHub repository:
```bash
git init
git add -A
git commit -m "Initial deploy"

# Create a private repo on github.com, then:
git remote add origin https://github.com/YOUR_USERNAME/gege-mould-website.git
git branch -M main
git push -u origin main
```

2. In Railway dashboard:
   - **New Project → Deploy from GitHub repo**
   - Select your repository
   - Railway auto-detects the Node.js app

### Option B: Deploy via Railway CLI

```bash
# Install Railway CLI
npm install -g @railway/cli

# Login
railway login

# Deploy
cd backend
railway init
railway up
```

## Step 6: Update Frontend Form Endpoints

Update the API base URL in the frontend JavaScript. Each form currently posts to a relative path like `/api/quote`. For the split deployment, they need to point to Railway.

**Option 1: Edit `js/main.js`** (the `API_BASE` variable)

Look for:
```js
var API_BASE = window.GEGE_API_BASE || '/api';
```

Change the fallback to:
```js
var API_BASE = window.GEGE_API_BASE || 'https://gege-mould-backend.up.railway.app/api';
```

Then **re-run `node build.js`** to regenerate `dist/` with the updated JS.

**Option 2: Use a production-only config in each page**

Add this `<script>` tag before the closing `</body>` in each page (or in `partials/scripts.html`):
```html
<script>window.GEGE_API_BASE = 'https://gege-mould-backend.up.railway.app/api';</script>
```

The build script will inject this into every page when you re-run `node build.js`.

## Step 7: Handle File Uploads on Railway

Railway's free tier includes **1 GB of ephemeral storage**. Uploaded RFQ files (STEP, PDF, etc.) are saved temporarily for email attachment purposes. This is sufficient for the expected traffic volume.

If you need persistent file storage later:
- Upgrade to Railway Hobby plan ($5/month) for persistent volumes
- Or use an S3-compatible service (Cloudflare R2, Backblaze B2)

## Free Tier Limits (Railway)

| Resource | Free Limit | Sufficient? |
|---|---|---|
| Runtime hours | 500 hrs/month | ✅ ~720 hrs/month needed — Upgrade to Hobby ($5/mo) if exceeded |
| RAM | 512 MB | ✅ Express + SQLite uses ~80 MB |
| Disk | 1 GB | ✅ Minimal file storage |
| Network | 100 GB/month | ✅ More than enough for form submissions |

**Note:** The free tier includes 500 hours/month. A month has ~720 hours. If the backend runs 24/7, you'll need the Hobby plan ($5/month). Alternatively, Railway spins down after inactivity — the first request "wakes" it up (cold start ~2-5 seconds). For a contact form, this is fine.
